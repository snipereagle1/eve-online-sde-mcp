# Task: SDE Download & Update Check

## Description

Implement startup network logic: HEAD check to detect new build, conditional download + extraction, ETag verification, old dir cleanup.

## Acceptance Criteria

- [x] HEAD `https://developers.eveonline.com/static-data/eve-online-static-data-latest-jsonl.zip` — follow redirect, parse build number from final URL
- [x] Build number regex: `eve-online-static-data-(\d+)-jsonl\.zip`
- [x] Compare against `meta.json` build (or absent → always download)
- [x] `--redownload` flag forces download even if build matches
- [x] Download with `reqwest` blocking + `indicatif` progress bar to stderr (bytes downloaded / total)
- [x] Save zip to temp file, verify Content-Length matches
- [x] Extract zip to `sde-{build}/` using `zip` crate; skip `__MACOSX/` entries
- [x] Parse `_sde.jsonl` from extracted dir → get `releaseDate`
- [x] Write `meta.json` with new build, release_date, etag
- [x] Delete old `sde-{prev_build}/` dirs after successful write
- [x] Network errors produce clear `anyhow` errors (no panics)

## Technical Details

URL pattern:
```
HEAD https://developers.eveonline.com/static-data/eve-online-static-data-latest-jsonl.zip
→ redirects to: .../tranquility/eve-online-static-data-{BUILD}-jsonl.zip
```

Parse build from redirect URL (reqwest follows redirects by default — use `.url()` on response or disable redirect and inspect `Location` header).

Progress bar: `indicatif::ProgressBar` with bytes style. Attach to `reqwest` response stream (blocking `Read` impl).
