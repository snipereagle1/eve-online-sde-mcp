# Task: Type & Item Tools + sde_status

## Description

Implement 6 MCP tools: `sde_status`, `sde_get_type`, `sde_search_types`, `sde_get_group`, `sde_get_category`, `sde_get_type_materials`.

## Acceptance Criteria

- [ ] `sde_status` — returns `{ build, release_date, data_dir, files_scanned, last_updated }` from SdeStore fields
- [ ] `sde_get_type` — input `{ type_id: u64 }`, fetch from `store.types`
- [ ] `sde_search_types` — input `{ query: String, published_only: Option<bool>, limit: Option<usize> }`, search `store.types.name_index`; filter `published: true` if `published_only` set; default limit 20
- [ ] `sde_get_group` — input `{ group_id: u64 }`, fetch from `store.groups`
- [ ] `sde_get_category` — input `{ category_id: u64 }`, fetch from `store.categories`
- [ ] `sde_get_type_materials` — input `{ type_id: u64 }`, fetch from `store.type_materials`
- [ ] All tools return `serde_json::Value` (full record as-is, language filtered)
- [ ] Unknown IDs return MCP tool error with `"ID {id} not found in {file}"`
- [ ] Input schemas registered with rmcp (type_id: integer, query: string, etc.)

## Technical Details

`published_only` filter: after fetching match records, filter where `value["published"] == true`. Do name_index scan first (fast), then fetch + filter.

Search result order: alphabetical by matched name, or by offset (insertion order). Limit applied after fetch.

Task 005 (MCP layer + fetch_by_id/search_by_name helpers).
