# Task: Map Tools + BFS Route

## Description

Implement 6 MCP tools: `sde_get_solar_system`, `sde_search_solar_systems`, `sde_get_region`, `sde_get_constellation`, `sde_get_route`, `sde_get_station`.

## Acceptance Criteria

- [ ] `sde_get_solar_system` — input `{ solar_system_id: Option<u64>, name: Option<String> }`, fetch by ID or name from `store.solar_systems`; error if both or neither provided
- [ ] `sde_search_solar_systems` — input `{ query: String, limit: Option<usize> }`, search `store.solar_systems.name_index`
- [ ] `sde_get_region` — input `{ region_id: Option<u64>, name: Option<String> }`, fetch from `store.regions`
- [ ] `sde_get_constellation` — input `{ constellation_id: u64 }`, fetch from `store.constellations`
- [ ] `sde_get_station` — input `{ station_id: u64 }`, fetch from `store.npc_stations`
- [ ] `sde_get_route` — input `{ from_system_id: u64, to_system_id: u64 }`, BFS over `store.stargate_graph`; returns `{ jumps: usize, path: Vec<u64> }`
- [ ] BFS runs on `tokio::task::spawn_blocking` thread
- [ ] BFS returns error if either system not in graph
- [ ] BFS same-system input → `{ jumps: 0, path: [id] }`
- [ ] BFS unreachable systems → MCP tool error "No route found"
- [ ] BFS worst-case < 500ms (K-space is ~5000 systems; wormhole space excluded from stargates)

## Technical Details

BFS implementation:
```rust
fn bfs_route(graph: &HashMap<u64,Vec<u64>>, from: u64, to: u64) -> Option<Vec<u64>> {
    if from == to { return Some(vec![from]); }
    let mut visited = HashSet::new();
    let mut queue = VecDeque::new();
    let mut prev: HashMap<u64,u64> = HashMap::new();
    queue.push_back(from);
    visited.insert(from);
    while let Some(cur) = queue.pop_front() {
        for &next in graph.get(&cur).unwrap_or(&vec![]) {
            if visited.insert(next) {
                prev.insert(next, cur);
                if next == to {
                    // reconstruct path
                }
                queue.push_back(next);
            }
        }
    }
    None
}
```

Stargate graph: bidirectional. Each stargate record has `systemID` + `destination.systemID`. Scan adds edge both directions.
