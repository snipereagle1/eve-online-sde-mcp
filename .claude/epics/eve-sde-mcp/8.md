# Task: Blueprint Tools

## Description

Implement 2 MCP tools: `sde_get_blueprint`, `sde_get_blueprint_for_product`.

## Acceptance Criteria

- [ ] `sde_get_blueprint` — input `{ blueprint_type_id: u64 }`, fetch from `store.blueprints`
- [ ] `sde_get_blueprint_for_product` — input `{ product_type_id: u64 }`, look up `store.product_to_blueprint` map, then fetch blueprint record; return `null` if no blueprint produces this type
- [ ] `product_to_blueprint` map populated correctly during scan (task 004) — verify with fixture
- [ ] Both tools return full blueprint record (all activities included)
- [ ] Unknown blueprint_type_id → MCP tool error
- [ ] product_type_id with no blueprint → `{ "result": null }` (not an error)

## Technical Details

Blueprint JSONL structure:
```json
{
  "blueprintTypeID": 692,
  "activities": {
    "manufacturing": {
      "products": [{ "typeID": 34, "quantity": 100 }],
      ...
    }
  }
}
```

Scan in task 004 extracts `activities.manufacturing.products[0].typeID` → maps to `blueprintTypeID`. Verify this assumption holds for reaction blueprints too (may use `activities.reaction`).

## Dependencies

Task 005 (MCP layer). Task 004 must populate `product_to_blueprint` (scan task).
