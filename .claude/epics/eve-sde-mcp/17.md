# Task: SDE Bump Workflow

## Description

Create `.github/workflows/sde-bump.yml` that runs weekly, detects a new SDE build, and opens a PR to update `PINNED_BUILD` in `src/sde_version.rs`.

## Acceptance Criteria

- [ ] `.github/workflows/sde-bump.yml` created
- [ ] Schedule: Mondays 06:00 UTC (`cron: '0 6 * * 1'`)
- [ ] Step: HEAD request to CCP's stable redirect URL → parse new build number from redirect URL
- [ ] Compare parsed build against `PINNED_BUILD` constant in `src/sde_version.rs`
- [ ] If newer: `sed`-update the constant, open PR titled `chore: bump CI SDE fixture to {build}`, labels `automated` + `sde-bump`
- [ ] Permissions: `contents: write`, `pull-requests: write`
- [ ] Dry-run or mocked HEAD response test documented (so the workflow can be validated without a real SDE release)
